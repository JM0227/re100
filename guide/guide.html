  • [사용법]
  • 검색 조건 2개 이하, 각 항목의 길이가 짧은 경우 사용한다.
  • table을 사용하며 각 영역의 너비는 폼 태그에서 직접 컨트롤하는 것이 아닌 col style을 통해 컨트롤한다.(col style=“width:110px”)

  • 태그의 너비 직접 삽입 금지(select style=“width:100px” 금지), 위에서 설명한 table의 col style을 통해 너비 조절할 것.

Search Form - Type2

  • [개요]
  • 버튼 영역을 뺀 나머지 영역의 너비는 버튼 영역을 뺀 나머지 영역을 100%로 사용.
  • 각 검색 항목은 고정 너비가 아니라 차지하는 영역에 따라 화면 사이즈 조절 시 유동적으로 움직인다.
  • [row/col 개념]
  • row = 행(table의 tr과 유사한 개념)
  • col = 열(table의 td와 유사한 개념)
  • 열(col)은 한 행(row) 내에 최대 4개까지만 배치하도록 하며 타이틀+폼 태그로 구성된 경우에는 dl 사용/폼 태그 단일로만 구성된 경우에는 div로 사용.
  • [col의 너비 조절]
  • col_* 클래스를 통해 너비 조절(col_1 ~ col_12)
  • col_1 = 한 행의 1/12 차지 , col_6 = 한 행의 50%(6/12) 차지, col_12 = 한 행의 100%(12/12) 차지.
  • 사용 예시 : div class="col col_12" or dl class="col col_12"
  • [col의 너비 조절(반응형)]
  • 3개의 반응형 중단점을 사용하며 중단점은 max-width 미디어쿼리 기반.
  • 중단점 1) max-width:1600px(1600px 미만에서 작동) / class="col_xxxl_*"(col_xxxl_1 ~ col_xxxl_12)
  • 중단점 2) max-width:1400px(1400px 미만에서 작동) / class="col_xxl_*"(col_xxl_1 ~ col_xxl_12)
  • 중단점 3) max-width:1200px(1200px 미만에서 작동) / class="col_xl_*"(col_xl_1 ~ col_xl_12)
  • 사용 예시 : dl class="col col_4 col_xxl_6 col_xl_12"(default값으로 4/12 영역 차지 -> 1400px 미만에서 6/12 차지 -> 1200px 미만에서 12/12 차지)
  • [form 태그 사용]
  • 태그의 width 직접 컨트롤 금지(select style="100px" 금지), class(col_*)로 width 조절.
  • 체크박스 3개 이상 또는 길이가 길 때는 Multi Select 사용을 권장한다.

Textarea

  • use class : select

  • 체크박스 여러개 사용
